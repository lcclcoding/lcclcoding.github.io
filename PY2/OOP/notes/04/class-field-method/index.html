    Unit 04: Class Fields and Methods

    This unit will introduce you to the difference between instance and class variables/methods

    Variables

    Instance Variables

    As a recap, instance variables are used to store data unique to each object. They are defined with self inside the __init__ constructor and belong to each specific instance of a class.

    Using the Rectangle class from Unit 01, we can see that the Rectangle class has 2 instance variables: base and height

    1
2
3
4
5
6
    class Rectangle:
    def __init__(self, base, height):
        self._base = base         # instance variable
        self._height = height     # instance variable

    ...

    Every Rectangle object instance has its own base and height, so changes to one object’s instance variables won’t affect others.

    Class Variables

    In contrast, class variables are shared among all instances of a class. They are typically used for properties that should be consistent for every object, or to keep track of class-level information.

    For example, all rectangles have four sides, and you may want to count how many rectangles have been created:

    class Rectangle:
    SIDES = 4               # class variable (constant)
    _count = 0              # class variable (for tracking)

    def __init__(self, base, height):
        self._base = base
        self._height = height
        Rectangle._count += 1    # update class variable

    ...

    No matter how many Rectangle objects you create, both SIDES and count are shared and accessible from every instance and through the class itself.

    Methods

    Instance Methods

    Like instance variables, instance methods operate on individual objects and can access or modify instance variables. In Python, they always take self as their first parameter.

    1
2

    class Rectangle:
    def __init__(self, base, height):
        self._base = base
        self._height = height

    def area(self):                         # instance method
        return self._base * self._height

    ...

    Class Methods

    Similarly, class methods are used for operations that affect the class as a whole, not specific instances. They are defined with the @classmethod decorator and take cls as their first parameter.

    class Rectangle:
    _count = 0

    def __init__(self, base, height):
        self._base = base
        self._height = height
        Rectangle._count += 1

    @classmethod
    def how_many(cls):              # class method
        return cls._count           # cls here is Rectangle

    Limited Access

    Class methods cannot access instance variables (since they don’t have access to self), but can access class variables and other class methods.

    Summary

    Summary Table

    Field/Method Type How to Define Belongs To First Parameter Can Access Use Case
    Instance variable self.base Instance - Instance Data unique to each object
    Instance method def area(self): Instance self Instance Behavior based on instance attributes
    Class variable count = 0 Class - Class Data shared by all instances
    Class method @classmethod Class cls Class Behavior related to the class itself
