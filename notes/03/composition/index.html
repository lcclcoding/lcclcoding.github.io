    Unit 03: Composition

    In Object-Oriented Programming (OOP), composition is another fundamental design principle: objects can be constructed from other objects. This allows you to build complex systems flexibly, by combining simpler, well-defined components.

    Creating Separate Classes for Engine and Brake

    A real car has an engine system and has a brake system. Instead of bundling these directly into the Car, we can create distinct classes for each. This enables modular design and reuse.

    The Engine class models an engine that can be started or stopped, and calculates acceleration based on how much you press the pedal.

    class Engine:

    def __init__(self, max_acceleration: float) -> None:
        self.max_acceleration: float = max_acceleration
        self.is_on: bool = False

    def start(self) -> None:
        self.is_on = True

    def stop(self) -> None: 
        self.is_on = False

    def is_running(self) -> bool:
        return self.is_on

    def apply(self, paddle_ratio) -> float:
        if paddle_ratio  1:
            raise Exception("Paddle ratio must be between 0 and 1.")

        if not self.is_running() and paddle_ratio > 0:
            raise Exception("The engine cannot accelerate when it is stopped.")

        return self.max_acceleration * paddle_ratio

    class Brake:

    def __init__(self, max_deceleration: float):
        self.max_deceleration: float = max_deceleration
        self.wear: float = 0                   

    def apply(self, paddle_ratio: float) -> float:
        if paddle_ratio  1:
            raise Exception("Paddle ratio must be between 0 and 1.")

        self.wear += 0.00001 * paddle_ratio    
        # Braking effect reduces as wear increases, and is always negative (slows the car)
        return self.max_deceleration * paddle_ratio * max(self.wear, 0.1) * -1

    class Car:
    def __init__(self, start_pos: float, engine: Engine, brake: Brake) -> None:
        self.position: float = start_pos
        self.speed = 0

        self.engine: Engine = engine
        self.brake: Brake = brake

    def start(self) -> None:
        self.engine.start()

    def stop(self) -> None:
        self.engine.stop()

    def _accelerate(self, acceleration: float) -> None:
        # Internal method: should not be called directly from outside this class
        dt = 1.0
        v0 = self.speed
        v1 = max(v0 + acceleration * dt, 0)
        dx = 0.5 * (v0 + v1) * dt

        self.speed = v1 
        self.position += dx

    def drive(self, paddle_ratio: float) -> None:
        acceleration = self.engine.apply(paddle_ratio)
        self._accelerate(acceleration)

    def brake(self, paddle_ratio: float) -> None:
        deceleration = self.brake.apply(paddle_ratio) # this is negative
        self._accelerate(deceleration)

    Unintended side effects

    When you pass objects in Python, you are passing references to the actual objects and not creating copies (unlike with primitive types like int, float, or str). Always be mindful of this, especially when composing objects as sharing objects (aliasing) can have unintended side effects if not managed carefully!
